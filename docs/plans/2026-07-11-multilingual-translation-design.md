# 多语言翻译设计

## 范围

桌面端从固定的“日文到简体中文”升级为可配置翻译方向。源语言默认自动检测，也可从常用语言中手动选择；目标语言必须明确选择。常用语言覆盖简体中文、繁体中文、英语、日语、韩语、法语、德语、西班牙语、俄语和葡萄牙语，并提供自定义语言名称与 BCP 47 代码入口。

## 数据流

语言配置属于翻译任务，不属于模型 Provider。React 保存 `sourceLanguage` 和 `targetLanguage`，调用 `translate_project` 时传给 Tauri。Tauri 将两者传入 `TranslationOrchestrator`，Provider 的结构化提示词同时包含源语言和目标语言。`auto` 会明确要求模型自行识别每段原文语言。

翻译结果保留本次语言配置。校对表头和导出摘要动态显示目标语言；导出命令携带目标语言代码，并用清理后的代码生成独立目录名，例如 `Game-zh-CN`。

## 边界

“可选择”表示模型请求支持该翻译方向，不表示游戏运行时一定能正确显示。第一版不实现字体覆盖、RTL 排版或每个语言组合的专项质量规则。控制码、占位符与独立目录导出规则保持不变。

## 测试

Provider 契约测试验证源语言和目标语言确实进入请求体。Tauri 测试验证语言参数进入编排器。React 工作流测试验证用户选择语言后，翻译调用、校对表头及导出调用使用同一配置。
