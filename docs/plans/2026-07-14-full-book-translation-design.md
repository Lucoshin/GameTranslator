# 一键翻译全书设计

## 目标

书籍项目概览提供“一键翻译全书”主操作。用户确认源语言、目标语言和模型后，只需一次操作即可翻译全部章节，不再逐章启动任务。

## 交互

- 入口位于书籍概览的书籍信息与进度区域，属于全书级操作。
- 任务运行时按钮禁用并显示“正在翻译全书…”，工作区顶部同步显示任务进行中。
- 未配置模型时打开统一模型配置抽屉，并明确提示需先配置模型。
- 章节编辑页保留“翻译本章”，用于局部重译。

## 数据流与安全策略

前端复用 `translate_book_project` 命令：具体章节传章节 ID，全书传 `null`。后端将空章节范围解释为全部章节，并按照目录顺序收集段落。全书任务跳过已有非空译文，避免覆盖人工修改和已校对内容；再次启动时自然从未译段落继续。逐章任务保持原有重译语义。

后端沿用现有批处理、并发策略、进度事件、QA 检查和项目持久化链路，因此全书翻译不是界面假循环，结果仍写回同一书籍项目文件。

## 验证

- 前端集成测试验证概览按钮发送全书范围。
- Rust 单元测试验证跨章节收集并跳过已有译文。
- 完整前端测试、Rust 测试、Clippy 与 Release 构建作为交付门禁。
