# 高吞吐翻译设计

## 基准

Mayfly 包含约 6,399 个对话块。当前固定 32 条一批、并发 1，共约 200 次串行网络等待。文件模板缓存后的重复扫描仅约 0.04 秒，模型请求已经是绝对主瓶颈。

## 调度

翻译核心按场景和字符预算构造批次，而不是只按条数。每批最多 64 条、约 24,000 UTF-8 字节，避免少量超长文本撑爆上下文。OpenAI-compatible 使用有界工作队列；均衡模式默认并发 8，极速模式 16。Ollama 默认并发 1，用户明确配置本地并行能力后才提高。

每个原始批次独立完成失败拆批和重试。HTTP 429 触发指数退避；连续限流时降低后续有效并发，成功窗口后逐步恢复。任何批次失败都不取消其他批次。

## 协议

稳定逻辑 ID 只留在本地。发送模型时映射为批内整数 `0..n`，响应返回后校验数量、顺序并恢复原 ID。这样避免模型为每条 Ren'Py 文本重复生成长路径和 translate 标识。

## 持久化

每个成功批次立即写入 SQLite。缓存键由原文、场景上下文、语言方向、模型、Provider 地址和提示词版本组成。重新运行时先加载精确命中，仅调度缺失项。任务异常退出后，下一次从未完成片段继续。

## UI

模型设置增加稳定、均衡和极速预设，也允许高级用户调整并发。进度使用“缓存命中、模型完成、失败、总数”真实计数。已完成批次持续写入当前任务状态，不等待全任务结束才更新。

## 安全

并发不改变占位符保护、结构化响应验证、失败拆批和导出 QA。缓存只接受完整通过占位符恢复的译文。不同模型、语言或提示词版本绝不共享缓存。
