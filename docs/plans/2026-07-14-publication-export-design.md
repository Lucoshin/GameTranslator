# 书籍出版导出与历史设计

## 目标

把当前仅支持 Markdown 的书籍导出升级为可交付的出版工作流，同时保持原始书籍只读。首版覆盖 Markdown、DOCX、EPUB 3 和印刷 PDF，并记录可恢复、可追溯的导出历史。

## 用户流程

“导出”阶段展示四张格式卡片。用户先补充作者、译者、出版社、ISBN、版权说明和可选封面，再选择格式。PDF 默认使用大32开（140 × 203 mm），并提供 A5、16开预设。每次导出通过原生保存对话框选择位置，成功后立即写入历史。

“历史”阶段按时间倒序展示格式、文件名、输出路径、目标语言、版式和导出时间。每条记录可以打开文件位置，或按相同格式与配置再次导出。删除历史只删除记录，不删除用户已经导出的出版文件。

## 格式规范

- Markdown：UTF-8，保留书名、出版元数据、章节和完整译文；未翻译段落回退为原文。
- DOCX：可编辑交稿；包含书名页、版权信息、章节分页，以及标题/章节/正文样式。正文采用首行缩进 2 字符和适合中文审校的行距。
- EPUB 3：标准 mimetype、container、OPF、nav、章节 XHTML 和 CSS；写入语言、作者、译者、出版社、ISBN 等元数据。
- PDF：印刷校样而非印厂制版文件；默认大32开，支持 A5/16开、页边距、装订线、章节另页、页码和页眉。嵌入可用中文字体，避免跨机器乱码。

## 架构

`content-book` 拥有与 UI 无关的出版模型和各格式写入器。Tauri 层负责保存对话框、应用数据目录中的历史 JSON、打开文件位置和命令 DTO。React 只负责编辑出版信息、选择格式/版式、展示成功结果与历史。

导出命令接收 `BookExportRequest { project, format, profile }`，返回 `BookExportRecord`。记录仅在文件写入成功后追加，避免出现指向不存在文件的历史。旧书籍 JSON 通过 serde 默认字段兼容加载。

## 错误与安全

- 永不覆盖源书；用户必须选择独立输出位置。
- 导出失败不写历史，并显示具体格式和文件错误。
- 未翻译段落保留原文，保证导出稿完整。
- PDF 找不到可嵌入中文字体时明确报错，不生成乱码文件。
- 历史删除不触碰导出文件。

## 验收

每个格式都有结构测试；DOCX、EPUB、PDF 还需解包或解析验证关键元数据、章节和正文。前端测试覆盖格式选择、可见导出按钮、成功记录与历史列表。最终执行前端全量测试、书籍 crate 测试、生产构建和 Release EXE 重建。
