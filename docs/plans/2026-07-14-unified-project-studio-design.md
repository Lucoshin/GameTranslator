# 统一项目工作室设计

## 目标

GameTranslator 不再把游戏翻译与书籍译校表现为两套相互跳转的应用。两类内容共享项目中心、应用导航、模型配置、语言设置、任务反馈和“概览—翻译—校对—导出”阶段；内容类型只决定编辑器内部的呈现方式。

## 信息架构

应用启动后直接显示项目中心。项目中心只有一个主要动作“导入项目”，并在相邻位置提供“选择游戏目录”和“选择书籍文件”两个明确入口。打开项目后，左侧常驻导航始终保持同一位置与视觉样式：项目、概览、翻译、校对、导出、历史。顶部始终显示项目类型、项目名称、保存/任务状态和当前模型。

游戏项目继续使用现有概览、任务进度、文本表格和补丁导出能力。书籍项目在相同外壳内使用章节树、书稿编辑器和段落检查器；“阅读编辑/逐段校对”只是编辑器视图切换，不再成为新的页面层级。术语与 QA 留在右侧检查器，避免与阶段导航竞争。

## 视觉方向

采用克制的编辑工作室风格：深墨色常驻侧栏、暖纸色内容面、朱红色单一强调色、统一的细线图标与 4/8 像素间距体系。游戏和书籍不再分别使用“启动海报”和“独立书房”两种视觉语言。内容区可以因类型呈现不同密度，但顶栏、导航、按钮层级、加载状态与错误反馈保持一致。

## 状态与返回行为

- 品牌按钮始终返回项目中心，不关闭或清空当前项目。
- 关闭项目后回到项目中心，并保留书籍列表与模型配置。
- 从项目中心重新打开当前项目时恢复其阶段与编辑状态。
- 项目中心、游戏项目、书籍项目均可直接打开模型配置。
- 每个页面只有一个主要操作：导入、开始翻译、确认校对或导出。

## 验收标准

- 首屏不再出现独立游戏宣传页。
- 游戏与书籍项目打开后能看到相同的应用侧栏和顶部项目栏。
- 两类项目均遵循“概览—翻译—校对—导出”的命名。
- 书籍编辑器不再出现第二套“书稿/编辑/问题/术语/导出”导航。
- 游戏现有扫描、翻译、校对、补丁导出与历史测试继续通过。
- 书籍现有导入、自动保存、按章翻译、校对与导出测试继续通过。
