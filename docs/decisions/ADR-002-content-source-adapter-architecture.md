# ADR-002：将翻译工作台与游戏引擎解耦

## 状态

已接受

## 日期

2026-07-12

## 背景

当前应用的翻译、QA、术语和补丁能力已经以稳定 Segment（片段）工作，但项目入口仍以 RPG Maker / Ren'Py 游戏目录为中心。这个边界会让星露谷物语、RimWorld 模组、书本和视频等新内容类型被迫伪装成“游戏引擎”，进而混淆扫描、上下文、安装和导出的责任。

目标是让翻译核心只处理可追踪文本片段和翻译任务；内容来源负责读取和上下文；输出目标负责安全写回或导出。原游戏与原始内容保持只读。

## 决策

新增抽象层时采用三段式管线：

```text
Content Source Adapter → Translation Workspace → Output Adapter
```

- **Content Source Adapter**：识别来源、提取稳定片段、提供预览与上下文，并声明可用输出能力。
- **Translation Workspace**：复用现有的 Segment、术语、翻译记忆、模型编排和 QA；不得依赖具体游戏、模组或文件格式。
- **Output Adapter**：以来源明确支持的方式生成补丁、语言包、字幕或文档副本；负责原始内容哈希、备份和安装/卸载策略。

现有 RPG Maker、Ren'Py 适配器将保留，逐步迁移为 `GameSourceAdapter` 与对应的 `GamePatchOutputAdapter`，而非删除或重写已验证的提取与回写逻辑。

## 内容类型与首个候选适配器

| 内容类别 | 候选来源适配器 | 安全输出 |
| --- | --- | --- |
| RPG Maker / Ren'Py 游戏 | 现有适配器迁移 | 独立补丁、受控安装与卸载 |
| 星露谷物语模组 | SMAPI 模组本地化资源适配器（待确认具体格式） | 独立语言包或模组副本 |
| RimWorld 模组 | XML / DefInjected 本地化适配器（待确认版本与模组约定） | 独立语言包或模组副本 |
| 书本 / 文档 | EPUB、Markdown、TXT 等文档适配器 | 翻译副本，不覆盖原文 |
| 视频 | 字幕文件与转写适配器 | SRT / VTT 等字幕副本；不承诺视频内嵌文字替换 |

“待确认”表示实现前必须基于用户提供的真实样本或官方格式契约验证，不能猜测字段或兼容多个臆测结构。

## 实施进度（2026-07-12）

- 已新增 `content-core`：定义来源类别、稳定来源元数据、通用 Segment 与输出能力声明。
- 已新增 `content-game`：以薄包装接入既有 RPG Maker / Ren'Py 提取逻辑，旧的 `engine-*` 调用路径仍可工作。
- 已新增 `content-rimworld`：基于最小 XML fixture，仅读取模组 `About/About.xml`、`Languages/English/Keyed` 与 `Languages/English/DefInjected` 的直接文本节点。
- RimWorld 当前可生成独立语言包（`Languages/ChineseSimplified/Keyed`、`DefInjected` 与 `LanguageInfo.xml`），已接入桌面端目录选择、扫描、翻译、导出与补丁历史。安装会备份已有同名语言文件；卸载仅恢复备份或删除仍匹配补丁哈希的文件。

## 备选方案

### 继续扩展 `EngineAdapter`

- 优点：短期改动少。
- 缺点：文档和视频没有“引擎”概念；模组安装、文档副本和字幕导出会污染游戏接口。
- 结论：拒绝。

### 直接实现动态第三方插件

- 优点：外部贡献者可自行发布适配器。
- 缺点：桌面端需要处理不受信任代码加载、版本兼容、签名与权限边界。
- 结论：当前拒绝；首版仍将适配器编译进应用。

### 仅保留通用文件翻译

- 优点：模型简单。
- 缺点：失去游戏事件、角色、模组定义和字幕时间轴上下文，且无法安全安装。
- 结论：拒绝。

## 迁移步骤

1. 在 `engine-core` 旁新增内容来源与输出契约，只复用现有 `Segment`，不改变翻译核心协议。
2. 将现有 RPG Maker / Ren'Py 的识别、提取和导出包装进新契约，并保持现有命令与补丁格式兼容。
3. 以真实星露谷或 RimWorld 模组样本建立适配器契约测试，再增加第一个模组适配器。
4. 增加文档、字幕适配器；视频仅在有可靠转写输入时处理字幕文本。
5. UI 将“选择游戏目录”演进为“选择内容来源”，按适配器声明展示可用的预览、导出、安装和卸载动作。

## 后果

- 翻译、QA 和模型能力可用于多个内容领域，避免复制业务逻辑。
- 每个适配器必须显式声明读写范围和安装能力，降低误改原文件风险。
- 星露谷、RimWorld、书本和视频尚未在本次变更中宣称支持；在没有真实契约测试前，界面不得展示为可用。
